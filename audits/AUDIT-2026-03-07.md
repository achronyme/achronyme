## BUGS (must fix before release)

### B1. UTF-8 column tracking bug in lexer
- **File:** `achronyme-parser/src/lexer.rs:616-618`
- **Issue:** Multi-byte UTF-8 chars (e, n, emoji) increment `col` once per byte instead of once per character. A 2-byte char like `e` reports col+2 instead of col+1.
- **Impact:** Diagnostic underlines misaligned for any non-ASCII source code.
- **Fix:** Track byte position and character position separately, or increment col per char not per byte.

### ~~B2. `running_sum.unwrap()` panic in Plonkish gadgets~~ — FALSE POSITIVE
- **File:** `compiler/src/plonkish_backend/gadgets.rs:388,482`
- **Reason:** `enforce_n_range` has an early return for `num_bits == 0` (fix from issue #59, line 335-339). The `unwrap()` on line 388 is only reachable when `num_bits >= 1`, guaranteeing the loop executes at least once. Similarly, `emit_is_lt_bounded` computes `num_bits = effective_bits + 1 >= 1`.

### B3. JSON fallback produces invalid JSON
- **File:** `cli/src/commands/mod.rs:85`
- **Issue:** The `unwrap_or_else` fallback for JSON serialization uses raw `format!()` without escaping quotes/backslashes in `diag.message`.
- **Impact:** Malformed JSON output when serialization fails.
- **Fix:** Use `serde_json::json!` in the fallback path too, or escape the message.

### B4. `circuit` command ignores `--error-format`
- **File:** `cli/src/commands/circuit.rs:40-51`, `cli/src/main.rs:45-56`
- **Issue:** The circuit command doesn't accept `ErrorFormat` — all errors render as human regardless of `--error-format`.
- **Impact:** Feature incomplete; JSON/short consumers can't use `circuit` command.
- **Fix:** Thread `ErrorFormat` through `circuit_command()` like the other commands.

### B5. Array index error reports `usize::MAX` instead of actual value
- **File:** `ir/src/lower/functions.rs:186`
- **Issue:** When `field_to_u64()` fails on an out-of-bounds index, `index` is set to `usize::MAX` before the bounds check, so the error says "index 18446744073709551615 out of bounds".
- **Impact:** Misleading error message.
- **Fix:** Report "array index too large" separately before the bounds check.

### B6. Parse errors lose line/col when converted to IrError
- **File:** `ir/src/lower/mod.rs:206,302`
- **Issue:** `IrError::ParseError(err.message)` discards line/col from the Diagnostic. When converted back via `to_diagnostic()`, defaults to `point(0,0,0)`.
- **Impact:** IR parse errors point to start of file instead of actual error location.
- **Fix:** Store the full Diagnostic or at least (line, col) in `IrError::ParseError`.

## WARNINGS (should fix)

### W1. Undefined variable uses wrong error variant
- **File:** `compiler/src/codegen.rs:144`
- **Issue:** When there's no span, undefined variables use `CompilerError::UnknownOperator` — semantically wrong.
- **Fix:** Use a dedicated variant or `DiagnosticError`.

### W2. `MissingOperand` variant is dead code
- **File:** `compiler/src/error.rs:22`
- **Issue:** Never constructed anywhere. Appears in match arms but never created.
- **Fix:** Remove the variant.

### W3. `_span` parameter unused in `load_module()`
- **File:** `ir/src/lower/module.rs:45`
- **Issue:** The span of the import statement is passed but discarded. Module errors (CircularImport, ModuleNotFound) have no source location.
- **Fix:** Add `OptSpan` to those IrError variants and use the span.

### W4. `to_diagnostic()` fallback span `(0,0,0)` for spanless errors
- **File:** `ir/src/error.rs:220`
- **Issue:** Multiple IrError variants default to pointing at line 0, col 0.
- **Fix:** Either propagate spans into those variants or omit the primary span.

### W5. Short format inconsistent with standard `file:line:col:` pattern
- **File:** `cli/src/commands/mod.rs:127-134`
- **Issue:** Outputs `line:col_start:col_end: severity: message` (no file, includes col_end). Standard tools use `file:line:col:`.
- **Fix:** Include file (or `-` for stdin) and drop col_end.

### W6. Levenshtein distance too permissive for short names
- **File:** `compiler/src/suggest.rs:39-40`
- **Issue:** `max_distance=2` means 67% error rate for 3-char names like "foo" — may suggest unrelated identifiers.
- **Fix:** Scale max_distance: e.g., `min(2, name.len() / 3)`.

### W7. `register hygiene` assert panics instead of returning error
- **File:** `compiler/src/function_compiler.rs:78`
- **Issue:** `assert_eq!(reg, self.reg_top - 1)` panics on internal bugs. Users see a panic backtrace.
- **Fix:** Return `CompilerError::InternalError` instead.

### W8. Module let binding errors lose module context
- **File:** `ir/src/lower/module.rs:113-114, 141-142`
- **Issue:** Errors from `lower_expr()` propagate without indicating which module/binding failed.
- **Fix:** Wrap with `.map_err()` adding module/binding context.

### W9. Empty array declarations not validated
- **File:** `ir/src/lower/mod.rs:162-199`
- **Issue:** `declare_public_array(name, 0)` silently creates an empty array.
- **Fix:** Validate `size >= 1`.

## NITS (nice to have)

### N1. Inconsistent error message capitalization
- **File:** `compiler/src/codegen.rs:134 vs 140`
- **Issue:** "undefined variable" (lowercase) with span vs "Undefined variable" (capital) without span.

### N2. Byte offset discarded in ParseError-to-Diagnostic conversion
- **File:** `achronyme-parser/src/error.rs:37`
- **Issue:** `SpanRange::point(err.line, err.col, 0)` always uses byte_offset=0.

### N3. No UTF-8 tests in diagnostic renderer
- **File:** `achronyme-parser/src/render.rs`
- **Issue:** All tests use ASCII only. The UTF-8 bug (B1) went undetected.

### N4. Error recovery span tests don't verify correctness
- **File:** `achronyme-parser/src/parser/tests.rs`
- **Issue:** `Stmt::Error { span }` nodes are created but spans are never inspected.

### N5. `get_const_value()` is O(n) per lookup
- **File:** `ir/src/lower/functions.rs:438-446`
- **Issue:** Linear scan of all instructions. Slow for large circuits.

### N6. Duplicate `span_to_json` functions
- **File:** `cli/src/commands/mod.rs:88-115`
- **Issue:** `span_to_json()` and `span_to_json_with_label()` differ only by a label field.

### N7. Suggestion test assertion is weak (OR condition)
- **File:** `compiler/src/codegen.rs:463-465`
- **Issue:** `!diag.suggestions.is_empty() || diag.message.contains("cout")` — passes even if suggestions are empty.

### N8. Module alias collisions silently shadow
- **File:** `ir/src/lower/module.rs:62-63`
- **Issue:** Re-aliasing a module doesn't check for name collisions.
