---
title: "Declaraciones"
description: "Declaraciones públicas y de testigo en circuitos."
---

Cada circuito necesita declaraciones de entrada explícitas. Estas le dicen al compilador qué variables son **públicas** (visibles para el verificador) y cuáles son **testigo** (conocidas solo por el probador). Las entradas se declaran como parámetros del bloque `circuit`.

## Declaraciones Escalares

Declara una variable a la vez en los parámetros del circuito:

```ach
circuit example(output: Public, secret: Witness) {
    assert_eq(secret, output)
}
```

O múltiples parámetros:

```ach
circuit example(x: Public, y: Public, a: Witness, b: Witness, c: Witness) {
    assert_eq(a + b + c, x + y)
}
```

Las entradas públicas se convierten en parte de la declaración pública de la prueba — cualquiera que verifique la prueba puede ver sus valores. Las entradas testigo permanecen privadas para el probador.

## Declaraciones de Array

Declara un array de entradas de tamaño fijo con sintaxis de corchetes:

```ach
circuit merkle(root: Public, leaf: Witness, path: Witness Field[3], indices: Witness Field[3]) {
    merkle_verify(root, leaf, path, indices)
}
```

Esto crea tres variables indexadas (`path_0`, `path_1`, `path_2`) a las que accedes con indexación estándar. Las declaraciones de array funcionan tanto para entradas públicas como de testigo:

```ach
circuit example(commitments: Public Field[4], secrets: Witness Field[4]) {
    // ...
}
```

## Uso de Entradas Declaradas

Una vez declaradas, las entradas son variables ordinarias. Úsalas en expresiones, pásalas a funciones integradas e indexa arrays:

```ach
circuit multiply(output: Public, a: Witness, b: Witness) {
    let product = a * b
    assert_eq(product, output)

    let sum = a + b
    assert_eq(sum, a + b)
}
```

Los elementos de array se acceden por índice:

```ach
circuit sum_check(expected_sum: Public, vals: Witness Field[3]) {
    let acc = vals[0]
    let acc = acc + vals[1]
    let acc = acc + vals[2]
    assert_eq(acc, expected_sum)
}
```

## Alternativa por CLI

Puedes pasar entradas con banderas del CLI:

```bash
ach circuit program.ach --inputs "a=3,b=5,output=15"
```

El compilador detecta automáticamente qué parámetros son públicos y cuáles son testigo a partir de la definición del circuito.

Las entradas de array se pasan con nombres indexados:

```bash
ach circuit merkle.ach --inputs "root=123,leaf=42,path_0=99,path_1=77,indices_0=0,indices_1=1"
```

## Anotaciones de Tipo

Los parámetros del circuito combinan visibilidad y tipo. Consulta [Anotaciones de Tipo](/es/circuits/type-annotations/) para detalles.

```ach
circuit example(root: Public Field, flag: Witness Bool, path: Witness Field[3], indices: Witness Bool[3]) {
    merkle_verify(root, flag, path, indices)
}
```

Anotar un testigo como `Bool` ahorra 1 restricción al omitir la verificación booleana.

## Errores

| Error | Causa |
|-------|-------|
| `DuplicateInput` | Mismo nombre declarado como `Public` y `Witness`, o declarado dos veces |
| `UndeclaredVariable` | Usar una variable que nunca fue declarada como `Public` o `Witness` |
| `AnnotationMismatch` | Tipo declarado no coincide con el tipo inferido |
