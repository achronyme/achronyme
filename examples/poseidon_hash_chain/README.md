# Poseidon hash chain — Achronyme workspace example

Proves knowledge of a preimage whose 5-round iterated
[circom Poseidon(2)](https://github.com/iden3/circomlib/blob/master/circuits/poseidon.circom)
hash equals a public digest, without revealing the preimage.

Uses the canonical circomlib template, so any commitment produced here
is interchangeable with one generated by tornado-cash, semaphore, or
any other circomlib-based protocol.

## Layout

```
poseidon_hash_chain/
├── achronyme.toml     project manifest + [circom] libs path
├── README.md          this file
└── src/
    └── main.ach       entry: off-chain chain + prove block
```

## Run

From the Achronyme workspace root (the directory that contains the
`test/circomlib/` checkout):

```sh
ach run examples/poseidon_hash_chain/src/main.ach
```

`ach` walks up from `main.ach`, finds `achronyme.toml`, and resolves
`circuits/poseidon.circom` through the `[circom] libs` list — no
explicit `--lib` flag needed.

Expected output:

```
=== Poseidon hash chain (circomlib) ===
Rounds:         5
Public digest:  <32-byte field element>

Hash-chain proof verified ✓
The verifier learned the digest but not the preimage.
```

## What the prove block constrains

```
h1 = Poseidon(preimage, 0)
h2 = Poseidon(h1, 0)
h3 = Poseidon(h2, 0)
h4 = Poseidon(h3, 0)
h5 = Poseidon(h4, 0)
h5 === final_hash       // public equality
```

`final_hash` is the only public input. `preimage` and every
intermediate are private witnesses.

## Why this matters

The circom template imported here (`Poseidon(2)` with `t=3`, full
rounds 8, partial rounds 57) is exactly what circomlibjs and every
production tornado/semaphore deployment uses. Achronyme routes that
template through the same `ProveIR → IR → R1CS → Groth16` pipeline as
native `.ach` code, so the verifier contract deployed on-chain is
bit-compatible with any circomlib-generated proof.
