## 1. Essence / Esencia

**EN:** Achronyme is a programming language that compresses verifiable truths. It bridges general-purpose code and zero-knowledge cryptography through a single, accessible syntax — letting developers prove computation inline, without leaving their program.

**ES:** Achronyme es un lenguaje de programación que comprime verdades verificables. Conecta código de propósito general con criptografía zero-knowledge a través de una sintaxis única y accesible — permitiendo a los desarrolladores probar computación inline, sin salir de su programa.

### Tagline

> **Timeless, compressed truths.**
>
> **Verdades atemporales, comprimidas.**

### Name / Nombre

"Achronyme" — from Greek *a-* (without) + *chronos* (time), blended with *acronyme* (French: acronym). A language that compresses truths beyond time: zero-knowledge proofs are verifiable regardless of when they were generated, by whom, or where. The proof stands on its own.

"Achronyme" — del griego *a-* (sin) + *chronos* (tiempo), fusionado con *acronyme* (francés: acrónimo). Un lenguaje que comprime verdades más allá del tiempo: las pruebas zero-knowledge son verificables sin importar cuándo se generaron, por quién, o dónde. La prueba se sostiene por sí sola.

## 6. Voice & Tone / Voz y Tono

### Principles / Principios

1. **Explain, don't gatekeep.** ZK is already intimidating. Every sentence should reduce confusion, not add jargon. / **Explica, no excluyas.** ZK ya es intimidante. Cada oración debe reducir confusión, no agregar jerga.

2. **Be direct.** Say what it does, not what it "aims to provide". / **Sé directo.** Di qué hace, no qué "busca proporcionar".

3. **Technical is fine, obscure is not.** Use technical terms when precise, but always in context. / **Técnico está bien, oscuro no.** Usa términos técnicos cuando sean precisos, pero siempre en contexto.

4. **Confident, not arrogant.** State capabilities clearly without overpromising. / **Confiado, no arrogante.** Declara capacidades claramente sin sobre-prometer.

### Examples / Ejemplos

| Context | Do / Hacer | Don't / No hacer |
|---------|-----------|-------------------|
| Feature description | "Prove any block of code inline with `prove {}`." | "Achronyme leverages cutting-edge cryptographic primitives to enable seamless proof generation." |
| Error message | "Division by zero in witness computation at line 12." | "An unexpected error has occurred during the constraint generation pipeline." |
| Docs intro | "Write code. Prove parts of it. That's it." | "Welcome to Achronyme, a next-generation zero-knowledge proof programming language designed to..." |
| Limitation | "Loops in circuits must have a known bound." | "Due to the nature of arithmetic circuits, iterative constructs require compile-time deterministic bounds." |

## 7. Logo / Logo

> **Status: pending design.** Guidelines below define constraints for when the logo is created.
>
> **Estado: pendiente de diseño.** Las guías abajo definen restricciones para cuando se cree el logo.

### Requirements / Requisitos

- Must work at `16px` (favicon) through `200px+` (hero)
- Must be legible in monochrome (single color on dark or light bg)
- Must have a standalone icon mark (for favicon, social avatars, CLI)
- Wordmark should use the project's monospace typeface or a custom letterform
- The icon mark should evoke **compression**, **verification**, or **timelessness** — not generic blockchain/crypto imagery (no chains, no locks, no shields)

### Clear Space / Zona de Exclusión

Minimum clear space around the logo: equal to the height of the letter "a" in the wordmark.

### Variants / Variantes

| Variant | Usage |
|---------|-------|
| Icon + Wordmark (horizontal) | Primary. Docs header, README. |
| Icon only | Favicon, social avatar, CLI badge. |
| Wordmark only | Inline text references, footer. |
| Monochrome (white on dark) | Default for dark backgrounds. |
| Monochrome (dark on light) | For light backgrounds, print. |

## 8. Applications / Aplicaciones

### Documentation Site

- Background: Void (`#0A0A0F`)
- Sidebar: Surface (`#13131A`)
- Active nav item: Proof Glow background + Proof text
- Code blocks: Surface background, Subtle border, Proof for keywords
- Links: Proof color, underline on hover

### CLI Output

- Success messages: Valid green (`#34D399`)
- Error messages: Error red (`#F87171`)
- Warnings: Warn amber (`#FBBF24`)
- Accent/highlights: Proof purple (`#8B5CF6`)
- Use monospace naturally (it's a terminal)

### GitHub / Social

- Repository social preview: Void background, centered icon mark, tagline in Text Secondary
- README badges: Proof purple background with white text
- Open Graph images: Void background, code snippet with syntax highlighting using brand colors

### Code Syntax Highlighting (for docs/marketing)
